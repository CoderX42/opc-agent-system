# OPC Agent System 技术栈、功能进度与后续规划

> 统计时间：2026-07-08  
> 项目定位：面向一人有限责任公司（OPC）的 AI Agent 数字员工系统，通过财务、客服、法务、行政等垂直 Agent 提供企业运营辅助能力。

## 1. 项目总体概况

OPC Agent System 当前采用 `pnpm workspace` 管理的前后端 Monorepo 架构，已形成 Web 前端、NestJS 后端、数据库迁移、种子数据、基础监控、鉴权、AI Provider 抽象、业务 Agent 模块等核心能力。

当前系统主线包括：

- **统一业务后台**：提供登录、仪表盘、Agent 办公室、财务、客服、法务、行政、系统设置等页面。
- **多 Agent 数字员工**：围绕财务、客服、法务、行政四类业务角色组织功能模块。
- **AI 能力接入层**：封装 DeepSeek、Ollama、OpenAI-Compatible、Anthropic 等 Provider，并支持按 Agent 配置模型。
- **企业数据管理**：基于 PostgreSQL + TypeORM 管理用户、Agent、财务、客服、法务、行政、知识库等实体。
- **工程化能力**：包含 Docker Compose、数据库迁移、E2E 测试、性能基线、接口文档、统一异常和响应封装。

   - 财务：强化发票 OCR 字段解析准确度，补充发票与交易自动关联规则。
   - 客服：完善 AI 回复建议、工单优先级自动分类和常见问题知识库匹配。
   - 法务：沉淀合同风险规则库，支持审查结果结构化存储和前端高亮展示。
   - 行政：补充任务提醒、日程冲突检测和会议纪要模板。

### 7.2 中期规划：增强 Agent 智能化与可运营性

优先目标：从“管理系统 + AI 调用”升级为“可配置、可观测、可运营的 Agent 工作台”。

1. **Agent 任务编排**
   - 引入任务队列或工作流状态机，统一管理 Agent 任务创建、执行、重试、取消、失败原因。
   - 将办公室可视化中的任务、进度、日志接入真实任务执行数据。

2. **RAG 能力升级**
   - 增加 Embedding 生成、向量库检索、相似度排序和引用来源。
   - 支持知识库文档上传、切片、索引、重建和权限隔离。

3. **工具调用能力**
   - 为财务、客服、法务、行政定义可调用工具，例如查发票、建工单、审合同、建日程。
   - 记录 Agent 工具调用轨迹，供前端审计和回放。

4. **模型路由策略**
   - 根据 Agent 类型、任务复杂度、成本和延迟选择模型。
   - 增加 Provider 健康检查、失败降级、超时控制和调用成本统计。

5. **前端体验升级**
   - 增加实时通知、任务进度推送、Agent 日志流、可拖拽任务队列。
   - 优化移动端适配，或正式创建 `packages/mobile` 作为独立移动端项目。

### 7.3 长期规划：生产化与商业化能力

优先目标：满足真实企业使用、部署、审计、扩展和运维要求。

1. **权限与租户体系**
   - 增加角色权限矩阵、资源级权限、操作审计。
   - 如面向多公司使用，扩展多租户隔离。

2. **生产部署能力**
   - 增加 Dockerfile、生产 Compose/Kubernetes 配置、健康检查、日志采集。
   - 将指标接入 Prometheus/Grafana，错误接入 Sentry 或同类系统。

3. **数据安全与合规**
   - 敏感字段加密、API Key 加密存储、上传文件访问控制。
   - AI 请求脱敏、审计日志、数据留存策略。

4. **外部系统集成**
   - 财务：对接发票平台、银行流水、会计软件。
   - 客服：对接网页客服、微信、邮件、企业 IM。
   - 法务：对接电子签、合同模板库、审批流。
   - 行政：对接日历、会议系统、文档协作平台。

5. **商业化配置能力**
   - Agent 模板市场、企业知识包、行业 Prompt 包。
   - 计费、额度、用量统计和成本分析。
