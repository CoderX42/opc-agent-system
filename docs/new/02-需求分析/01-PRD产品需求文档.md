# PRD 产品需求文档

> 版本 v1.0

## 一、产品概述

OPC Agent Platform 是面向未来企业和一人公司的数字员工操作系统,它不是聊天机器人平台,而是一家公司,用户可以通过自然语言创建属于自己的 CEO Agent、行政 Agent、客服 Agent、法务 Agent、财务 Agent、开发 Agent,等等。

每个 Agent 都是一个长期存在的数字员工,拥有自己的身份、知识、记忆、工具和成长路径。

## 二、产品目标

- 降低 AI 使用门槛,让普通用户也能拥有自己的 AI 员工团队
- 实现多 Agent 协同工作,而不是单一聊天
- 构建企业级的长期记忆和知识沉淀,避免企业知识随着人员流动而消失
- 打造可视化、模块化、可扩展的数字员工生态,最终形成 Agent Marketplace

## 三、用户角色

- 个人用户
- 独立开发者
- 创业者
- 企业用户(包括中小企业和软件公司)
- 平台管理员

## 四、用户使用流程

注册 → 创建组织 → 进入数字办公室 → 用户输入"我想创建一个 NestJS 高级开发工程师",系统自动解析,生成职位职责、Prompt、知识库、工具、工作流和蓝图 → 创建 Agent 成功 → 进入 Agent Studio 继续培养 → 开始工作 → 多 Agent 协同 → 任务完成 → 知识沉淀

## 五、功能结构

- **用户中心**:注册登录、组织管理、订阅支付
- **Agent Center**:创建、编辑、删除、分组、标签、权限和蓝图
- **Agent Chat**:聊天、工具调用、文件上传、知识引用、协同记录
- **Knowledge Hub**:知识库管理,文档、网页、GitHub、Notion、飞书导入
- **Memory Center**:长期记忆、用户画像、项目历史、经验沉淀
- **Workflow Center**:流程设计、多 Agent 协同、审批流
- **Tool Center**:GitHub、邮箱、浏览器、HTTP、数据库、Docker、MCP
- **Dashboard**:公司驾驶舱,今日任务、Agent 状态、成本、日志
- **Desktop**:桌面助手,通知、快捷操作、数字员工
- **后台管理**:租户、用户、模型、费用、日志

## 六、MVP 范围

- **P0**:用户系统、Agent 管理、聊天、知识库、记忆、工作流、工具调用、Dashboard、桌面端
- **P1**:Agent Studio、Agent Forge、移动端

## 七、非功能需求

- **性能**:普通查询响应三秒内、支持异步任务和横向扩展
- **安全**:多租户隔离、RBAC 权限、审计日志、数据加密
- **可维护性**:插件化架构、模块解耦、完整 API 文档

## 八、一句话总结

PRD 的目标不是描述一个聊天式 AI,而是完整定义一个可以运行数字员工团队的企业级操作系统。
